# 네이버 GFA 리포팅 중계 플랫폼 — 사용 가이드 (운영자용, v2)

> 웹 버전(보기 좋음): 아티팩트 링크로 공유. 이 문서는 저장소 보관용.
> 운영자 콘솔: `naver-gfa.vercel.app/admin` · 광고주 API: `naver-gfa.vercel.app/v1`

---

## 01. 이 플랫폼은 무슨 일을 하나요?

네이버는 GFA API 키를 **개별 광고주가 아니라 미디어렙사에게만** 발급합니다. 렙사 키를 광고주에게
그대로 주면 **A 광고주가 B·C·D 등 모든 광고주 데이터까지** 보게 됩니다.

이 플랫폼은 그 사이에서 **중계(브로커)** 역할을 합니다:

```
광고주 A·B·C  --우리 발급 전용 키-->  우리 플랫폼(중계)  --네이버 키(비공개)-->  네이버 GFA
                                    각 키는 담긴 광고계정 데이터만 조회
```

**핵심 원칙 — 네이버 키는 절대 광고주에게 나가지 않습니다.** 광고주는 우리 키로 우리 API만
호출하고, 시스템이 그 키에 담긴 계정 데이터만 걸러서 돌려줍니다.

## 02. 핵심 개념 — "키 관리 그룹"

좌측의 각 항목은 **키 관리 그룹**입니다. 하나의 그룹은:

- **여러 광고계정을 담을 수 있고(스코프)**,
- 광고주에게 줄 **시크릿(=API 키) 하나**를 가집니다.

그룹에 담긴 계정 데이터만 그 키로 조회됩니다. 이 구조로 두 경우를 모두 처리합니다:

| 경우 | 방법 |
|---|---|
| **계정 단위 키** (광고주 = 계정 1개) | 계정 1개만 담긴 그룹 → **개별 키 발급** |
| **묶음 키** (한 광고주에 계정 여러 개) | 여러 계정을 한 그룹에 **추가** |

> 왜 계정 단위가 기본인가 — “(넥서스팀) 식품_꿈비”, “(넥서스팀) 제약_ZPT”처럼 실제 광고주가
> 다른 계정이 많습니다. 계정 단위로 발급하면 데이터가 정확히 분리됩니다.

## 03. 운영자 콘솔 사용법 (`/admin`)

**A. 접속** — `/admin` → 우측 상단에 관리자 토큰 입력 → 접속.

**B. 계정 단위 키 발급 (가장 흔함)** — 아무 그룹 선택(없으면 임시 생성) → “계정 검색·추가”에
계정명/번호 검색 → 그 줄의 **[개별 키 발급]** → 그 계정만 담긴 그룹이 즉시 생성되고 시크릿 표시 →
**[복사]** 해서 광고주에게 전달.

**C. 묶음 키 (한 광고주의 여러 계정)** — 좌측 하단 “새 그룹 이름”에 이름 입력(예: 넥슨) → [생성]
→ “계정 검색·추가”에서 계정들을 [추가]로 담기 → 키 하나로 담긴 모든 계정 조회.

**D. 계정 제거 / 시크릿 재발급** — “이 키의 광고계정”에서 [제거]. 시크릿 유출 시 [시크릿 재발급]
(새 시크릿 발급, 이전 것 즉시 무효).

**E. 그룹 삭제 / 키 폐기** — 좌측 목록에서 hover 시 **[×]** 로 그룹 완전 삭제. 또는 상세에서
**[키 폐기]**(무효화하되 기록 보존).

**F. 사용 현황** — 상단 [사용 현황] 탭 → 오늘·7일 호출, 활성 키, 에러율, 일별 차트, 키별 사용량,
최근 로그. 기간 토글(14/30/90일)·CSV 내보내기. 개별 그룹 사용량은 관리 탭에서 그룹 선택 시 미니 차트.

> 같은 이름의 그룹은 중복 생성되지 않습니다(알럿).

## 04. 광고주가 데이터를 받아가는 법

광고주(개발자)에게 발급한 시크릿과 함께 아래를 전달:

```
# 공통 — 발급 키를 헤더에
Authorization: Bearer ngfa_xxxxxxxx.xxxxxxxxxxxx

# 내 광고계정 목록(이 키에 담긴 계정)
GET https://naver-gfa.vercel.app/v1/accounts

# 캠페인 성과 리포트 (account_no 생략 시 담긴 전체)
GET https://naver-gfa.vercel.app/v1/reports?date_from=2026-06-01&date_to=2026-06-30&account_no=4987
```

응답 예시:
```json
{
  "data": [
    { "stat_date":"2026-06-20", "naver_account_no":4987, "campaign_id":104012,
      "impressions":273083, "clicks":749, "cost":389595.17, "conversions":0 }
  ],
  "data_freshness": "2026-07-21T09:00:00+09:00"
}
```

| 필드 | 의미 |
|---|---|
| impressions | 노출 수 |
| clicks | 클릭 수 |
| cost | 광고비(원) |
| conversions | 전환 수 |
| data_freshness | 데이터 마지막 갱신 시각 |

> 데이터는 전일까지, 매일 새벽 갱신. 조회는 한 번에 최대 31일.

## 05. 꼭 알아둘 점

- **네이버 키·관리자 토큰은 외부 공유 절대 금지** — 팀 내부만.
- **키 발급 전 그룹에 담긴 계정 확인** — 한 그룹에 다른 회사 계정이 섞이면 그 회사 데이터가 노출.
- **시크릿은 발급/재발급 순간 한 번만 표시** — 놓치면 재발급.
- **책임은 렙사** — 네이버는 제3자 위임 이슈에 관여하지 않으므로, 광고주와 데이터 이용 동의를 받아둘 것.

## 06. 운영/한도 참고

- 네이버 관리계정 한도 **60회/분**. 수집기는 **활성 키에 담긴 계정만** 분당 60회 이하로 순차 수집.
- 계정명이 “(미보강)”이면 좌측 유지보수의 **전체 계정 이름 보강** 실행(권한 없는 일부 계정 제외).

## 07. 문제 해결(FAQ)

- **광고주 401** → 키 오타/폐기/재발급됨. 상태 확인 후 재발급.
- **데이터 비어있음** → 그룹에 계정이 담겼는지 + 해당 기간 실제 집행 여부 확인.
- **다른 광고주 데이터 노출?** → 불가(키에 담긴 계정만, 계정 기준 이중 방어). 단 담을 때 주의.
- **한 광고주에 계정 여러 개** → 한 그룹에 [추가] 반복. 키 하나로 전체 조회.
- **같은 이름 그룹 생성 안 됨** → 중복 방지. 다른 이름을 쓰거나 기존 그룹을 사용.
