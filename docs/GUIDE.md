# 네이버 GFA 리포팅 중계 플랫폼 — 사용 가이드 (운영자용)

> 웹 버전(보기 좋음): 아티팩트 링크로 공유. 이 문서는 저장소 보관용.
> 운영자 콘솔: `naver-gfa.vercel.app/admin` · 광고주 API: `naver-gfa.vercel.app/v1`

---

## 01. 이 플랫폼은 무슨 일을 하나요?

네이버는 GFA API 키를 **개별 광고주가 아니라 미디어렙사에게만** 발급합니다. 렙사 키를 광고주에게
그대로 주면 **A 광고주가 B·C·D 등 모든 광고주 데이터까지** 보게 됩니다.

이 플랫폼은 그 사이에서 **중계(브로커)** 역할을 합니다:

```
광고주 A·B·C  --우리 발급 전용 키-->  우리 플랫폼(중계)  --네이버 키(비공개)-->  네이버 GFA
                                    각 광고주는 자기 계정 데이터만 조회
```

**핵심 원칙 — 네이버 키는 절대 광고주에게 나가지 않습니다.** 광고주는 우리 키로 우리 API만
호출하고, 시스템이 자기 계정 데이터만 걸러서 돌려줍니다.

## 02. 제공 기능

- **광고주별 전용 키** — 광고주마다 스코프(허용 계정)가 박힌 키 발급·폐기.
- **스코프 리포팅 API** — 광고주가 자기 성과(노출·클릭·광고비·전환)를 직접 조회.
- **운영자 콘솔** — 광고주 등록, 계정 검색·배정, 키 관리를 웹 화면에서.
- **자동 데이터 수집** — 매일 새벽 네이버에서 계정·성과 자동 갱신.

## 03. 운영자 콘솔 사용법 (`/admin`)

**A. 접속** — `/admin` 접속 → 우측 상단에 관리자 토큰 입력 → 접속.

**B. 신규 광고주 추가** — 왼쪽 "광고주" 칸 하단 입력란에 이름 → [추가].

**C. 계정 배정** — 광고주 클릭 → "계정 검색·배정"에 이름/번호 검색 → [배정]. 여러 개면 반복.
> ⚠️ 배정 전 확인: 자동 생성 광고주는 접두로 묶여 드물게 다른 회사가 섞일 수 있음(계열사). 키 발급 전 훑어보기.

**D. 계정 해제** — "배정된 광고계정"에서 해당 줄 [해제].

**E. API 키 발급** — 광고주 선택 → 배정 계정 확인 → [새 키 발급] → **초록 박스에 한 번만** 표시되는 키를 복사해 광고주에게 전달.
> ⚠️ 키는 이 순간 딱 한 번만 보임. 놓치면 폐기 후 재발급.

**F. API 키 폐기** — 키 목록에서 [폐기] → 확인. 즉시 무효(401), 되돌릴 수 없음.

**G. 전체 계정 이름 보강** — 유지보수 칸 [전체 계정 이름 보강]. 네이버에서 계정명 일괄 채움(수 분).

## 04. 광고주가 데이터를 받아가는 법

광고주(개발자)에게 발급한 키와 함께 아래를 전달:

```
# 공통 — 발급 키를 헤더에
Authorization: Bearer ngfa_xxxxxxxx.xxxxxxxxxxxx

# 내 광고계정 목록
GET https://naver-gfa.vercel.app/v1/accounts

# 캠페인 성과 리포트 (account_no 생략 시 배정 전체)
GET https://naver-gfa.vercel.app/v1/reports?date_from=2026-06-01&date_to=2026-06-30&account_no=4987
```

응답 예시:
```json
{
  "data": [
    { "stat_date":"2026-06-20", "naver_account_no":4987, "campaign_id":104012,
      "impressions":273083, "clicks":749, "cost":389595.17, "conversions":0 }
  ],
  "data_freshness": "2026-07-21T09:00:00+09:00"
}
```

| 필드 | 의미 |
|---|---|
| impressions | 노출 수 |
| clicks | 클릭 수 |
| cost | 광고비(원) |
| conversions | 전환 수 |
| data_freshness | 데이터 마지막 갱신 시각 |

> 데이터는 전일까지, 매일 새벽 갱신. 조회는 한 번에 최대 31일.

## 05. 꼭 알아둘 점

- **네이버 키·관리자 토큰은 외부 공유 절대 금지** — 팀 내부만.
- **키 발급 전 배정 계정 확인** — 특히 자동 생성 광고주(계열사 병합 가능성).
- **키는 한 번만 표시** — 분실 시 폐기 후 재발급.

## 06. 문제 해결(FAQ)

- **광고주 401** → 키 오타/폐기됨. 콘솔에서 키 상태 확인, 재발급.
- **데이터 비어있음** → 계정 배정 여부 + 해당 기간 실제 집행 여부 확인.
- **다른 광고주 데이터 노출?** → 불가(키에 묶인 계정만, 이중 방어). 단 배정을 잘못하면 안 됨.
- **계정명 "(미보강)"** → 전체 이름 보강 실행. 권한 없는 일부 계정은 번호로 관리.
- **한 광고주에 계정 여러 개** → C단계 반복 배정. 키 하나로 전체 조회.
